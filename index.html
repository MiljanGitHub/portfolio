projects

email client - spring boot

The last project I did is an Email client, which is relaying on Spring technology on backend and Android on frontend. Our application is utilizing Java Mail API (for sending/receiving emails and for communicating with email servers) which was integrated in Spring environment, also, we used Hibernate ORM to communicate and persist data to MySQL database. One of the key assignments, for which I was responsible, was to enable authentication and authorization using Spring Security. Both of them were necessary in order for every User to identify itself to the server side and at the same time a specific User with a certain Role can have permission for some URLs and actions defined for his role. I implemented authorization using JWT tokens. Slf4j was used to log important events to the console.

email client - android

This project was coupled with project in Spring Boot. On this project my team and I provided a fronted for an Email client. Communication (i.e. hitting appropriate RESTful services on backend) was implemented using Retrofit3 library. Main focus of this projects was to provide to users a clean and consistent display of all Activities and Fragments used in app. Some of the biggest challenges were to display Email structure and organize Rules for dispatching emails to appropriate Folders.

web cinema

This web application is relaying on Java’s Servlet technology were HTTP server, Tomcat, was responsible for processing all HTTP requests. The app was simulating workflow of a Cinema. Users were able to pick Projection of certain a Movie, buy Ticket, view their profile, register themselves and cancel Ticket etc. Admins were having full CRUD operations on each entity. HTTP Session was used to allow/restrict usage of resources. Any change on any entity was persisted in SQLite database. The fronted of an application was done using JSPs and HTML5.

sales and distribution network

Java console application which simulates production and distribution of products from factories to sales outlets within a company. Assignment was to make a simulation of production and sales network of electronic devices by respecting basic principles of object oriented programming. Some of the entities in this app were: Factory, Company, Employee (with few subclasses), Outlet, Product, Transport etc. Important events were logged to txt file.

virtual classroom

Management and scheduling desktop application for creating Appointments across multiple Institutions in different Classrooms, where Admins, Professors and Assistants (TAs) were able to (un)schedule Appointments. Application was equipped with Telerik's WPF Schedule View Control. Any kind of CRUD operation was persisted to SQL Server. Application was written in C# using .NET’s WPF framework for desktop applications.

restaurant chain

Assignment for this app was to simulate workflow of a Restaurant Franchise. Like any other franchise, the User could make an Order of different Items, or to request for a Delivery. The Franchise had multiple Restaurants, on different Locations. On the end of each month, Admins were able to request for statistical Report of revenues of a Franchise. The emphasis of this console Java application was to properly organize objects and to apply core principles of object oriented programming. Any CRUD operation, on each entity, was recorded to a text file.

employees management system

Small Java console application which simulates Employers and their Employees. Demonstrates usage of: modeling objects, reading/writing from/to text files, Strategy design pattern, creating UML diagram.
